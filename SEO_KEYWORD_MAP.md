# 피서콕 SEO 키워드 맵

비브랜드 검색어에서도 노출되도록 설계한 키워드 전략과, 키워드별로 연결한 실제 페이지를 정리한 문서예요. Google Search Console·네이버 서치어드바이저에 실제 검색어 데이터가 쌓이면 이 문서를 기준으로 title/설명/본문을 다시 조정하세요.

## 1. 키워드 분류

### 브랜드 키워드
- 피서콕, PiseoKok, 피서콕 앱

브랜드 키워드는 이미 자연 유입(직접 검색)이 가능한 영역이라 별도 랜딩페이지를 만들지 않았고, 전 페이지의 title 끝에 "- 피서콕"을 붙이는 방식으로만 노출을 유지했어요.

## 2. 키워드 → 페이지 매핑표

| 목표 키워드(예시) | 연결 페이지 | 페이지 성격 |
|---|---|---|
| 한산한 해수욕장, 덜 붐비는 해수욕장, 사람 없는 해수욕장 | `/quiet-beaches` | 큐레이션 목록 |
| 한산한 계곡, 사람 적은 계곡 | `/quiet-valleys` | 큐레이션 목록 |
| 서울 근교 계곡/피서지, 당일치기 피서지 | `/near-seoul` | 거리순 목록 |
| 아이와 가기 좋은 계곡, 가족 피서지 추천 | `/family` | 큐레이션 목록 |
| 주차 편한 해수욕장, 해수욕장 주차 어디가 편한지 | `/parking-easy` | 큐레이션 목록 |
| 부산 해수욕장 추천 | `/regions/busan` | 지역 목록 |
| 강원도 한산한 해수욕장 | `/regions/gangwon` | 지역 목록 |
| 제주도 물놀이 장소 | `/regions/jeju` | 지역 목록 |
| 피서지 언제 가야 덜 붐비는지, 몇 시에 가야 덜 붐비나요, 주말에 덜 붐비는 여행지 | `/guide/best-time` | 가이드(essay) |
| 피서지 혼잡도, 계곡 사람 많은지, 혼잡도는 어떻게 계산 | `/guide/crowding` | 가이드(essay) |
| 해수욕장 혼잡도, 오늘 갈 만한 피서지, 주말 해수욕장 혼잡도 | `/` (홈), `/ranking` | 실시간성 콘텐츠 |
| {장소명} 혼잡도/주차/날씨 (예: 경포해수욕장 혼잡도) | `/spot/:id` (44곳) | 장소 상세 |
| 전체 피서지 목록, 지역/유형별 필터 | `/spots` | 필터 목록(canonical 고정) |

반려견 동반 관련 검색어는 전용 페이지를 만들지 않고 `/family` 페이지 FAQ에서 "확인된 데이터 없음 + 현장 확인 권장"으로만 답했어요 (5번 항목 참고).

## 3. 확인·갱신 프로세스

1. Google Search Console과 네이버 서치어드바이저에 `https://piseocok.vercel.app`을 등록하고 위 `sitemap.xml`을 제출하세요.
2. 매달 "실적 보고서(검색어별 노출수·클릭수·평균 게재순위)"를 확인해서, 위 매핑표에 없는데 실제로 유입되는 검색어가 있으면 이 문서에 추가하세요.
3. 특정 페이지의 평균 게재순위가 낮은데 노출수는 높다면, 그 페이지의 title/H1/본문 첫 문단을 검색어에 더 가깝게 다시 쓰세요 (키워드 반복이 아니라 실제 문장 톤 조정).
4. 신규 피서지 데이터가 늘어나면 `npm run build`의 prebuild 단계(`scripts/generate-sitemap.ts`)가 `MOCK_SPOTS` 기준으로 sitemap을 자동 재생성해요. TourAPI로 들어오는 전국 데이터까지 sitemap에 반영하려면 별도 배치 작업이 필요해요 (5번 항목 참고).

## 4. 생성·수정한 SEO 관련 파일

- `src/components/Seo.tsx`, `src/components/JsonLd.tsx` — title/meta/canonical/OG 및 JSON-LD 렌더링
- `src/utils/schema.ts` — BreadcrumbList, FAQPage, TouristAttraction JSON-LD 빌더
- `src/utils/withSeo.tsx` — 정적 콘텐츠 컴포넌트에 Seo를 씌우는 헬퍼(prerender 대응)
- `src/content/seoRoutesMeta.ts` — 10개 신규 페이지의 title/description/canonical 경로
- `src/content/curatedSpots.ts` — 한산함/거리/편의시설 기준 큐레이션 로직
- `src/content/disclaimers.ts` — 혼잡도·가족/반려동물·정보 갱신 관련 정직한 문구
- `src/pages/seo/*.tsx` — 신규 10개 랜딩/가이드 페이지
- `src/pages/HomePage.tsx`, `SpotDetailPage.tsx`, `SpotsPage.tsx`, `RankingPage.tsx`, `MapPage.tsx`, `FavoritesPage.tsx`, `MyPage.tsx` — Seo 적용
- `scripts/generate-sitemap.ts`, `scripts/prerender.ts`, `tsconfig.scripts.json` — sitemap 자동 생성 및 정적 프리렌더
- `public/robots.txt`, `vercel.json` — 크롤링 규칙, SPA 라우트 fallback

## 5. 데이터가 없어 반영하지 못한 항목

- **반려동물 동반 가능 여부**: 장소별 실제 정책 데이터가 없어 전용 페이지/뱃지를 만들지 않았어요. FAQ에서 "확인된 데이터 없음, 현장 확인 필요"로만 안내해요.
- **아이 동반 안전 검증**: `/family` 페이지의 선정 기준은 "편의시설 보유 + 낮은 예상 혼잡도"일 뿐, 수심·안전요원 등을 현장 검증한 것은 아니에요.
- **날씨(수온·바람·자외선)**: 연동된 데이터 소스가 없어 상세페이지의 "현재 기온"만 제공하고 나머지는 다루지 않았어요.
- **TourAPI 동적 장소의 sitemap 포함**: 현재 sitemap은 `MOCK_SPOTS`(큐레이션 44곳) 기준으로만 생성돼요. TourAPI로 들어오는 전국 데이터까지 포함하려면 빌드 시점에 TourAPI를 호출해 URL 목록을 만드는 별도 배치가 필요해요.
- **OG 이미지**: 정확한 1200×630 소셜 공유 이미지가 없어 앱 아이콘(`icon-512.png`)으로 대체했어요. 전용 디자인 이미지 제작을 권장해요.
